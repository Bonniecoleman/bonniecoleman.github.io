Resource Code: Link

Abstract

This project deals with a binary classification problem using a dataset containing a variety of categorical features with different levels of complexity and cardinality. The core objective is to build a robust preprocessing pipeline that can accurately transform each feature into a usable format through one-hot, ordinal, or binary encoding while preserving essential information. Aerial Cactus Identification

Resource Code: Link

Abstract

The project focuses on classifying aerial images to detect the presence of cacti using a Convolutional Neural Network (CNN). The dataset contains 17,500 labeled training images and 4,000 test images, all with a resolution of 32 x 32 pixels.

Main Features

  • Applied random padding, horizontal/vertical flips, rotations, and normalization to improve model robustness
  • CNN Architecture:
    • Five convolutional layers with Batch Normalization and LeakyReLU activation
    • Pooling layers for down-sampling and Average Pooling before flattening
    • Fully connected layers for final binary classification
  • Training Process: Used Adamax optimizer with a learning rate of 0.00006 and CrossEntropyLoss for 70 epochs
  • Evaluation Metrics: Used ROC-AUC to assess the model's binary classification performance
  • Private Leaderboard Score: 0.9998

Development Environment

  • Programming Language: Python
  • IDE: Jupyter Notebook
  • Libraries: PyTorch, torchvision, pandas, matplotlib, scikit-learn, OpenCV

Predict Future Sales

Resource Code: Link

Abstract

This project forecasts monthly sales based on historical daily sales data from a Russian software firm(1C Company). The workflow emphasizes capturing time-series characteristics by aggregating data at the monthly level and applying various preprocessing steps.

Main Features

  • Data Preprocessing:
    • Identified and removed outliers in price and sales volume to stabilize training
    • Standardized shop IDs by merging names that appeared as duplicates due to slight variations
  • Time-Series Structure:
    • Organized data by month (0–33) and treated the 34th month (November 2015) as the test set
    • Created lag features for the past three months of sales, average price, and transaction counts
  • Derived Variables:
    • Included the month a product was first sold (first_sold_month) to identify newly introduced items
    • Calculated metrics like sales change rates and average lagged sales to capture trends
  • Feature Engineering:
    • Converted categorical columns such as city and major_category with label encoding for the model
    • Used downcasting on numeric columns to reduce memory usage, given the large data size
  • Training Process:
    • Employed LightGBM, specifying categorical features like city and shop_id to reflect time-series factors
    • Used months 0–32 for training, month 33 for validation, and month 34 for testing
    • Clipped predictions between 0 and 20 and optimized for RMSE
  • Evaluation Metrics: RMSE was used to measure how closely the predicted sales matched actual figures
  • Private Leaderboard Score: 0.87442 (top 10.2%)
  • Programming Language: Python
  • IDE: Jupyter Notebook
  • Libraries: pandas, NumPy, scikit-learn, LightGBM, seaborn, matplotlib
  • Dataset: Kaggle
